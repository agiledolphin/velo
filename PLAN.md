# Velo / 微落开发计划

本文件记录产品阶段、交付范围与验收条件。具体架构决策见
[`docs/architecture.md`](docs/architecture.md)，已完成的版本变化见
[`CHANGELOG.md`](CHANGELOG.md)。

## 当前状态

- 当前版本：`0.1.0 Preview`
- 当前阶段：第一阶段已完成
- 下一阶段：接入真实媒体解析引擎

## 第一阶段：桌面外壳与解析闭环

状态：已完成（2026-07-13）

目标：建立可运行、可测试的桌面应用骨架，让用户能够提交视频页面地址并查看模拟解析结果。

- [x] 使用 Tauri 2、React、TypeScript 和 Rust 初始化桌面项目
- [x] 使用 Bun 管理前端依赖与脚本
- [x] 建立 `domain`、`application`、`infrastructure`、`commands` 分层
- [x] 实现 URL 校验与空闲、解析中、错误、成功状态
- [x] 实现 Mock 媒体引擎与浏览器预览 fixture
- [x] 展示媒体摘要、可用格式和格式选择
- [x] 完成 Velo / 微落首版界面与自定义可拖动标题区域
- [x] 增加前端与 Rust 单元测试
- [x] 初始化 Git，并推送 `main` 到 GitHub

验收结果：前端测试、Rust 测试和生产构建通过；应用尚不执行真实解析或下载。

## 第二阶段：真实媒体解析

状态：待开始

目标：使用 yt-dlp 替换 Mock 引擎，返回真实、稳定且可诊断的媒体信息。

- [ ] 设计并实现受限的外部进程执行器
- [ ] 接入 `YtDlpEngine`，保持现有 `MediaEngine` 接口稳定
- [ ] 管理 yt-dlp 可执行文件的定位、版本与平台差异
- [ ] 解析并规范化标题、站点、时长、封面与格式信息
- [ ] 支持超时、取消、无权限、站点不支持等错误
- [ ] 为真实引擎边界增加单元测试与集成测试
- [ ] 在引入远程资源前配置明确的内容安全策略

验收条件：用户提交受支持的页面地址后，可以看到真实媒体信息与可用格式；失败时能够获得清晰、可恢复的错误提示。

## 第三阶段：下载与文件处理

状态：规划中

目标：完成从格式选择到本地文件落盘的核心下载链路。

- [ ] 设计下载任务与进度事件模型
- [ ] 接入保存位置选择和文件命名规则
- [ ] 支持下载进度、速度、剩余时间与取消
- [ ] 按需接入 FFmpeg 进行音视频合并或转封装
- [ ] 处理覆盖、磁盘空间不足和文件权限错误
- [ ] 为未完成任务提供安全清理机制

验收条件：用户可以选择格式和保存位置，完成一次可观察、可取消、结果可定位的真实下载。

## 后续方向

以下能力在核心下载链路稳定后再评估优先级：

- 下载队列与历史记录
- 批量地址处理
- Cookie 与需要登录的网站支持
- 应用设置、日志导出与问题诊断
- yt-dlp、FFmpeg 和应用本体的更新策略
- Windows 与 Linux 的完整打包和发布验证

## 计划维护约定

- 阶段开始前确认目标、非目标和验收条件。
- 完成任务后更新复选框；范围变化应同时更新相关产品或架构文档。
- 已交付内容写入 `CHANGELOG.md`，不要只保留在 Git 提交信息中。
