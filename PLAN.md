- [x] 建立 `domain`、`application`、`infrastructure`、`commands` 分层
- [x] 实现 URL 校验与空闲、解析中、错误、成功状态
- [x] 实现 Mock 媒体引擎与浏览器预览 fixture
- [x] 展示媒体摘要、可用格式和格式选择
- [x] 完成 Velo / 微落首版界面与自定义可拖动标题区域
- [x] 增加前端与 Rust 单元测试

## 第二阶段：真实媒体解析

状态：已完成（2026-07-14）

目标：使用 yt-dlp 替换 Mock 引擎，返回真实、稳定且可诊断的媒体信息。

- [x] 将 `MediaEngine` 与 Tauri 解析命令调整为异步边界
- [x] 设计并实现受限的外部进程执行器
- [x] 接入 `YtDlpEngine`，保持现有 `MediaEngine` 接口稳定
- [x] 管理 yt-dlp 的下载、定位、固定版本、校验和与平台差异
- [x] 按目标三元组准备并通过 Tauri sidecar 纳入 yt-dlp
- [x] 构建并验证包含 yt-dlp 的 macOS 应用包
- [x] 建立 Windows、Linux 原生应用包构建与 sidecar 内容验证工作流
- [x] 在推送后的 CI 中取得并记录 Windows、Linux 成功结果
- [x] 解析并规范化标题、站点、时长、封面与格式信息
- [x] 通过受限 Rust 获取器显示真实封面并提供失败回退
- [x] 支持引擎缺失、超时、输出超限、异常响应和无格式错误
- [x] 支持解析取消并终止对应的 yt-dlp 进程
- [x] 细分登录、权限、地区、限流、不支持、不可用与网络错误
- [x] 使用固定 JSON fixture 覆盖真实引擎边界的单元测试
- [x] 增加显式启用、由调用方提供地址的真实站点集成测试入口
- [x] 在受控地址上运行并记录各平台兼容性结果
- [x] 为生产与开发环境配置分离且最小化的内容安全策略

验收结果：Tauri 运行时已使用 `YtDlpEngine`。引擎通过固定安全参数执行 yt-dlp，规范化单视频 JSON，并最多返回 24 个有效格式。每次解析使用独立请求 ID，用户取消后会释放解析 Future 并终止对应子进程；已知站点失败会映射为有限错误码，不暴露原始 stderr。生产 WebView 默认拒绝远程资源。构建钩子会按目标三元组准备已校验的 sidecar；macOS `.app` 已验证，Windows x64 NSIS 与 Linux x64 DEB 已通过原生发布验证。GitHub Actions 运行 `29313750794` 使用授权的 `x.com` 页面在 macOS arm64、Windows x64 与 Linux x64 上均返回一致的时长和 5 个有效格式。

验收条件：用户提交受支持的页面地址后，可以看到真实媒体信息与可用格式；失败时能够获得清晰、可恢复的错误提示。

## 第三阶段：下载与文件处理

状态：已完成（2026-07-16）

目标：完成从格式选择到本地文件落盘的核心下载链路。

- [x] 设计下载任务与进度事件模型
- [x] 接入保存位置选择和文件命名规则
- [x] 支持真实下载、进度、速度、剩余时间与取消
- [x] 接入 FFmpeg，为仅视频格式自动补充最佳音轨并合并封装
- [x] 通过受限流地址从约 1 秒处提取代表帧，并保留站点封面失败回退
- [x] 随应用打包受限 Deno runtime，并支持用户主动选择受限 Cookie 文件
- [x] 增加版本化设置入口，将 YouTube Cookie 限定为按站点、按需使用并支持持久配置
- [x] 增加内存态多任务页签，每个页签独立保留地址、解析结果、格式选择与下载进度
- [x] 限制同时解析与同时下载各最多 2 个，支持明确排队、取消等待与空位自动补位
- [x] 增加最多 50 条成功记录的本地下载历史，支持查看保存位置与二次确认清空
- [x] 支持系统或自定义默认下载目录、一键下载、单次另存为与重名自动编号
- [x] 完成 YouTube Cookie、解析取消、长标题信息展示与多页签切换的真实桌面回归
- [x] 处理覆盖、磁盘空间不足和文件权限错误
- [x] 为未完成任务提供安全清理机制

回归记录：2026-07-15 已在 macOS 桌面开发模式验证 YouTube Cookie 按需解析、
解析即时取消与迟到结果隔离、长标题下的格式信息展示、多页签状态保留，以及解析与
下载双队列的并发限制、取消等待和空位自动补位。

2026-07-16 已在 macOS 桌面开发模式验证系统及自定义默认下载目录、一键下载、单次
另存为、重名自动编号、目录设置持久化和成功下载历史。

验收结果：核心下载链路已支持多任务排队、可观察进度、即时取消、安全暂存与清理、
音视频合并、默认目录和单次另存为。前端、Rust 与 Clippy 在 Ubuntu 快速 CI 运行
`29443465917` 中全部通过；macOS 桌面真实解析、下载和设置持久化回归通过。

验收条件：用户可以选择格式和保存位置，完成一次可观察、可取消、结果可定位的真实下载。
