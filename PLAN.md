- 当前版本：`0.1.0 Preview`
- 当前阶段：第二阶段进行中
- 下一里程碑：运行跨平台 CI、完成第二阶段验收并提交

## 第一阶段：桌面外壳与解析闭环

状态：已完成（2026-07-13）

目标：建立可运行、可测试的桌面应用骨架，让用户能够提交视频页面地址并查看模拟解析结果。

- [x] 使用 Tauri 2、React、TypeScript 和 Rust 初始化桌面项目
- [x] 使用 Bun 管理前端依赖与脚本
- [x] 建立 `domain`、`application`、`infrastructure`、`commands` 分层
- [x] 实现 URL 校验与空闲、解析中、错误、成功状态
- [x] 实现 Mock 媒体引擎与浏览器预览 fixture
- [x] 展示媒体摘要、可用格式和格式选择
- [x] 完成 Velo / 微落首版界面与自定义可拖动标题区域
- [x] 增加前端与 Rust 单元测试
- [x] 初始化 Git，并推送 `main` 到 GitHub

验收结果：前端测试、Rust 测试和生产构建通过；应用尚不执行真实解析或下载。

## 第二阶段：真实媒体解析

状态：进行中（2026-07-14 启动）

目标：使用 yt-dlp 替换 Mock 引擎，返回真实、稳定且可诊断的媒体信息。

- [x] 将 `MediaEngine` 与 Tauri 解析命令调整为异步边界
- [x] 设计并实现受限的外部进程执行器
- [x] 接入 `YtDlpEngine`，保持现有 `MediaEngine` 接口稳定
- [x] 管理 yt-dlp 的下载、定位、固定版本、校验和与平台差异
- [x] 按目标三元组准备并通过 Tauri sidecar 纳入 yt-dlp
- [x] 构建并验证包含 yt-dlp 的 macOS 应用包
- [x] 建立 Windows、Linux 原生应用包构建与 sidecar 内容验证工作流
- [ ] 在推送后的 CI 中取得并记录 Windows、Linux 成功结果
- [x] 解析并规范化标题、站点、时长、封面与格式信息
- [x] 支持引擎缺失、超时、输出超限、异常响应和无格式错误
- [x] 支持解析取消并终止对应的 yt-dlp 进程
- [x] 细分登录、权限、地区、限流、不支持、不可用与网络错误
- [x] 使用固定 JSON fixture 覆盖真实引擎边界的单元测试
- [x] 增加显式启用、由调用方提供地址的真实站点集成测试入口
- [ ] 在受控地址上运行并记录各平台兼容性结果
- [x] 为生产与开发环境配置分离且最小化的内容安全策略

当前进展：Tauri 运行时已使用 `YtDlpEngine`。引擎通过固定安全参数执行 yt-dlp，规范化单视频 JSON，并最多返回 24 个有效格式。每次解析使用独立请求 ID，用户取消后会释放解析 Future 并终止对应子进程；已知站点失败会映射为有限错误码，不暴露原始 stderr。生产 WebView 已默认拒绝远程资源。构建钩子会按目标三元组准备已校验的 sidecar，macOS `.app` 已验证，Windows/Linux 原生 CI 工作流已建立；尚待推送后取得跨平台成功结果并记录真实站点兼容性矩阵。

验收条件：用户提交受支持的页面地址后，可以看到真实媒体信息与可用格式；失败时能够获得清晰、可恢复的错误提示。

## 第三阶段：下载与文件处理

状态：规划中

目标：完成从格式选择到本地文件落盘的核心下载链路。

- [ ] 设计下载任务与进度事件模型
- [ ] 接入保存位置选择和文件命名规则
- [ ] 支持下载进度、速度、剩余时间与取消
- [ ] 按需接入 FFmpeg 进行音视频合并或转封装
- [ ] 处理覆盖、磁盘空间不足和文件权限错误
- [ ] 为未完成任务提供安全清理机制

验收条件：用户可以选择格式和保存位置，完成一次可观察、可取消、结果可定位的真实下载。
