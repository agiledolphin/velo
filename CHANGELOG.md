# 变更日志

本文档记录 Velo / 微落的重要版本变化。版本号遵循[语义化版本](https://semver.org/lang/zh-CN/)，格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)。

## Unreleased

### 修复

- 修复取消解析按钮在状态切换后被同一次点击重新提交表单的问题；取消现在会使当前解析代次失效，迟到的结果不能再覆盖“解析已取消”状态。
- 修复超长媒体标题撑宽结果卡片并把格式容器、文件大小挤出可视区域的问题；标题改为稳定省略并保留完整悬停提示。
- 上述取消与长标题修复已在 macOS 桌面开发模式使用真实 YouTube 页面完成回归验证。

### 调整

- 增加最多 8 个视频任务的页签工作区；每个页签独立保留地址、解析结果、画质选择与下载状态，支持快捷键新建、关闭和标准方向键导航。
- 切换页签不会中断解析或下载；解析中的页签关闭时自动取消请求，下载中的页签需先取消任务后再关闭，避免产生无主后台下载。
- 固定解析工作区高度，画质列表改为内部滚动，并为下载状态与操作按钮预留稳定区域，避免格式数量或下载进度引起界面跳动。
- 增加可扩展的独立设置页和顶部设置入口，支持快捷键 `Cmd/Ctrl + ,`，返回主界面后保留当前解析状态。
- YouTube Cookie 改为版本化持久设置，提供按需、始终和停用三种模式，以及文件选择、更换、状态检查与清除。
- Cookie 参数按站点和单次请求决定；默认先匿名解析，仅在 YouTube 身份验证失败时携带 Cookie 重试，并沿用到对应媒体的预览与下载。
- 将应用标识由 `app.velo.desktop` 调整为 `org.azuresky.velo`，设置和平台应用数据目录随之使用新标识。

## 0.2.0 - 2026-07-14

### 新增

- 增加受限异步进程执行器，要求绝对可执行文件路径并禁止 Shell 命令拼接。
- 增加进程超时、输出上限和结构化执行错误。
- 增加进程输出捕获、超时终止、超限拒绝和 Shell 元字符安全测试。
- 增加 `YtDlpEngine`，解析并规范化 yt-dlp 单视频 JSON 输出。
- 增加固定 JSON fixture，覆盖字段映射、格式筛选、参数约束和错误映射。
- 增加 yt-dlp 路径定位：支持 `VELO_YT_DLP_PATH`、应用同目录和开发目录。
- 固定 yt-dlp `2026.07.04`，增加 macOS、Windows 与 Linux 平台资源清单和官方 SHA-256 校验和。
- 增加带下载大小上限、临时文件和失败回滚的原子引擎安装脚本，以及独立版本校验命令。
- 增加 `dev:desktop` 启动前校验和显式启用的真实站点集成测试入口。
- 增加解析请求注册与取消命令，取消后释放任务并终止对应的 yt-dlp 子进程。
- 增加取消按钮、取消反馈以及防止旧请求结果覆盖新请求的前端状态保护。
- 增加请求 ID 校验、取消 Future 释放和任务注册清理测试。
- 增加生产与开发环境分离的 CSP：发布包默认拒绝远程资源，开发环境仅开放本机 Vite 热更新所需来源。
- 增加 CSP 回归测试，防止生产策略引入通配符、远程来源、内联脚本或动态执行。
- 增加目标三元组到官方 yt-dlp 资产的映射，以及符合 Tauri 命名约定的 sidecar 准备脚本。
- 开发和发布构建前自动复用或下载固定版本、验证 SHA-256，并将 yt-dlp 作为 `externalBin` 纳入应用包。
- 增加 Tauri sidecar 配置与目标文件名回归测试。
- 增加 yt-dlp 站点失败分类：登录、权限、地区限制、请求限流、不支持地址、内容不可用和网络错误。
- 增加错误分类与原始 stderr 隔离测试，避免向界面暴露本地路径或引擎诊断细节。
- 增加 Windows x64 NSIS 与 Linux x64 DEB 原生 CI，运行测试、Clippy、应用构建、sidecar 内容检查并保留短期构建产物。
- 跨平台 Rust 测试前先准备目标架构 sidecar，确保 Tauri 构建脚本可以验证 `externalBin`。
- Windows x64 NSIS 与 Linux x64 DEB 已在 GitHub Actions 运行 `29291541333` 中通过原生打包、sidecar 内容检查和产物上传。
- 增加手动三平台真实站点兼容性工作流，待测地址仅从 GitHub Actions Secret 读取，日志不输出原始地址或媒体标题。
- 增加真实站点测试范围说明与兼容性矩阵文档。
- 固定版 yt-dlp 已在 GitHub Actions 运行 `29313750794` 中通过 `x.com` 页面的 macOS arm64、Windows x64 与 Linux x64 真实解析验证，三端均返回 5 个有效格式。
- 增加下载任务 ID、不可变任务描述、下载进度和生命周期事件领域模型。
- 下载事件使用稳定的 camelCase 序列化契约和递增序号，覆盖排队、开始、进度、处理、完成、取消与失败状态。
- 增加 Tauri 原生保存对话框，用户可按所选媒体格式选择目标文件。
- 增加跨平台安全文件名建议、绝对目标路径、扩展名匹配和 Windows 保留名校验。
- 下载任务增加不可变目标路径；保存位置验证通过后在界面显示，但当前不会提前启动下载。
- 增加受限封面获取器，将 yt-dlp 返回的远程封面转换为 WebView 可展示的本地数据 URL。
- 封面请求限制公网 HTTP/HTTPS、重定向次数、超时、图片 MIME 和 5 MiB 响应上限，并拒绝内网、本机、带凭据地址和 SVG。
- 结果卡片显示真实视频封面，加载中保留轻量状态提示，缺失或失败时自动回退至 `VELO` 占位图。
- 增加约 1 秒处代表帧生成：首次解析缓存最高 480p 视频流，FFmpeg 通过输入侧快速定位在受限协议与请求头范围内直接输出 JPEG；成功后替换站点封面，失败时静默保留原有回退。
- 固定并打包 Deno `2.9.2`，让 yt-dlp 使用随附 EJS 完整处理 YouTube JavaScript challenge，同时继续禁止远程组件。
- 增加用户主动选择 Netscape Cookie 文件的错误恢复入口；文件限制为 5 MiB，仅在当前运行内共享给解析、预览与下载任务。
- 修复 Cookie 文件选择窗口被 Tauri capability 拦截的问题，并保持文件对话框权限为最小的打开与保存范围。
- 增加受限 `YtDlpDownloader`，按用户选定格式和绝对目标路径执行真实媒体下载。
- 增加单任务协调器、重复任务保护、流式进度事件和真实子进程取消。
- 下载界面显示已下载大小、速度、剩余时间、确定或不确定进度条，以及完成、取消和失败状态。
- 下载先写入目标目录内的独立任务暂存区，成功后才替换目标文件，避免已有文件被跳过后误报完成。
- 取消或失败后清理完整暂存目录，并细分磁盘空间不足、写入权限、目标占用和清理失败提示。
- 增加显式启用的真实下载与真实取消集成测试；固定 X 页面最低清晰度格式已完成约 27 MiB 落盘和首个进度后取消验证。
- 增加按目标平台固定资产与 SHA-256 的 FFmpeg sidecar，并与 yt-dlp 一同纳入开发和发布构建。
- macOS、Linux 与 Windows 使用经压缩包与二进制双重校验的原生 FFmpeg 8.1.2；构建资源包含第三方来源与许可提示。
- 仅视频格式会自动选择最佳音轨，下载完成后由 FFmpeg 合并封装，并向界面发送处理状态。
- 固定 X 页面 `hls-109` 格式已完成真实视频、音频下载与合并验证，最终 MP4 同时通过视频流和音频流检查。

### 调整

- 将日常 CI 与跨平台发布打包拆分：非文档 push/PR 运行 Ubuntu 快速验证，完整 Windows/Linux 打包仅由版本标签、手动触发或打包关键文件变化触发。
- 为快速 CI、发布打包和真实站点兼容性工作流增加按平台隔离的 Cargo 缓存。
- 将 `MediaEngine`、应用状态和 Tauri 解析命令调整为异步接口，为 yt-dlp 接入做准备。
- 将 Tokio `1.52.3` 声明为直接依赖，仅启用进程、时间、异步 IO、运行时和测试宏所需功能。
- Tauri 运行时由 Mock 引擎切换为真实 `YtDlpEngine`；浏览器开发预览继续使用只读 fixture。
- 更新界面文案，移除“模拟解析结果”和过期的阶段提示。
- 将构建产物上传动作升级至 `actions/upload-artifact@v7`，使用当前 Node.js 24 运行时。

### 限制

- 兼容性矩阵目前只覆盖一个经授权的 `x.com` 页面，其他站点仍需逐项验证；默认测试不会访问网络。
- 取消下载可能保留 `.part` 临时文件，安全清理将在后续里程碑完成。

## 0.1.0 - 2026-07-13

首个 Preview 版本，完成桌面应用骨架和模拟解析闭环。

### 新增

- 基于 Tauri 2、React 19、TypeScript 7 和 Rust 的桌面项目。
- Bun 依赖管理、开发、测试和生产构建脚本。
- Rust 领域模型、应用接口、Tauri 命令和 Mock 媒体引擎。
- URL 校验以及空闲、解析中、错误和成功状态。
- 媒体摘要、模拟格式列表与格式选择界面。
- 浏览器开发模式下的只读预览 fixture。
- 前端 URL 单元测试和 Rust 媒体引擎测试。
- 产品说明与第一阶段架构文档。

### 调整

- 完成 Velo / 微落首版品牌界面和 Slogan 排版。
- 使用自定义顶部区域替代传统标题栏，并支持窗口拖动。
- 将默认窗口尺寸设为 `1120 × 660`，最小尺寸设为 `920 × 600`。
- 将结果区域改为随解析状态变化的紧凑布局。
- 锁定中英文 Slogan 的左侧位置，避免随窗口宽度移动。

### 限制

- 当前使用模拟引擎，不发起真实网络解析或视频下载。
- 尚未接入 yt-dlp、FFmpeg、Cookie、下载历史和自动更新。
