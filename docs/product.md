# Velo 第一阶段产品说明

## 单一目标

让用户粘贴一个视频页面地址，并清楚地看到 Velo 解析出的媒体信息与可用格式。

## 用户流程

1. 用户粘贴完整的 HTTP 或 HTTPS 地址。
2. Velo 在本地校验地址。
3. Rust 后端通过媒体引擎解析页面。
4. 界面展示标题、站点、时长和格式。
5. 第一阶段明确提示下载功能尚未启用。

## 验收标准

- 应用窗口可以启动并适配窄窗口。
- 输入框支持键盘操作并具有清晰焦点状态。
- 空地址、错误地址和非 Web 协议会被拒绝。
- 解析过程具有明确的进行中状态。
- Mock 引擎返回稳定、结构化的媒体信息。
- 用户可以选择一个模拟格式。
- 界面尊重系统的“减少动态效果”设置。

## 非目标

- 不进行真实网络解析或视频下载。
- 不读取浏览器 Cookie。
- 不接入 yt-dlp、FFmpeg 或 SQLite。
- 不实现批量任务、历史记录或自动更新。
